Test Overview

What is AFP08?

AFP08 is a CRO (Conversion Rate Optimisation) A/B test variation for conference.financialprofessionals.org. The variation injects a modal that promotes AFP 2026 conference registration. The modal fires automatically after the user has spent 15 continuous seconds on the site — across any page. The timer is stored in sessionStorage so that navigating between pages does not reset it.

Variation files

  • local_testing/Local2/variation/vB.js — Injects modal HTML, manages timer logic, handles close events, fires VWO analytics event
  • local_testing/Local2/variation/vB.css — Modal styling; uses min-width: 1024px media query to hide modal on all mobile/tablet devices

Design from AFP08.png

  • Dark modal (#070103 body, #241022 review strip) with AFP 2026 branding
  • Headline: "Why people attend AFP 2026" in yellow (#fc0)
  • Conference image with rounded corners (border-radius 70px)
  • 3 feature cards: See what's actually working / Compare approaches / Bring back better decisions
  • Two CTA buttons: Register Now (yellow filled) and View Program & Pricing (yellow outline)
  • Review quote from Cassie Wang, Head of Finance, Lightship Security Inc.
  • Close via X button (top-right) or clicking the background overlay
Note — background blur: When the modal shows, .mm-page receives filter: blur(3.5px) via the CSS rule html body.cre-t-8-show-modal > .mm-page. The overlay element itself has no background colour (it was commented out in the CSS), meaning the blur is the only visual cue for the background state.

Timer Logic

The timer is implemented entirely via sessionStorage and setTimeout:

// On first page load — stores epoch ms as string sessionStorage.setItem("startTime", String(Date.now())); // On every subsequent page (same tab, same session) const startTime = parseInt(sessionStorage.getItem("startTime"), 10); const elapsedTime = Date.now() - startTime; const remainingTime = Math.max(0, 15000 - elapsedTime); if (remainingTime === 0) { showModal(); // 15s already elapsed — show immediately } else { setTimeout(showModal, remainingTime); // schedule remaining time }
  • Cross-page persistence: sessionStorage survives same-domain navigation within a browser tab. Each new page that runs the variation reads the existing startTime and calculates the remaining countdown from there.
  • One-shot flag: Once the modal fires, sessionStorage.setItem("modalTriggered", "true") is written. Future variation executions check isExitPopupTriggered() and exit early — the modal will not fire again in the same session.
  • Desktop-only activation: The CSS media query @media (min-width: 1024px) gates the display of both .cre-t-8-modal-overlay and .cre-t-8-modal-container. Even if cre-t-8-show-modal is added to body on a mobile device, the elements remain hidden.

Test strategy note

Rather than waiting a real 15 seconds per test, the suite manipulates sessionStorage.startTime directly to simulate elapsed time. Setting startTime = Date.now() - 16000 before injecting the variation causes remainingTime to calculate as 0 — triggering the modal immediately. This keeps the full 150-test run under 4 minutes while testing identical code paths to real user behaviour.

The AFP site is mocked via page.route("https://conference.financialprofessionals.org/**") returning a minimal HTML shell. This avoids bot-protection, CSP issues, and network flakiness while still exercising the variation on the correct origin (same-origin sessionStorage required for navigation tests TC-07 and TC-08).
