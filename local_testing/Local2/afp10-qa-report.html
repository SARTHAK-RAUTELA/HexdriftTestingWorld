What testing was done

This QA run used Playwright (automated browser testing) against mock pages that replicate the AFP header's #global-login structure. Both the Control and Variation JavaScript + CSS files were injected into the mock pages to simulate VWO execution. Tests verified:

  • DOM injection accuracy (body class, button element, position, count)
  • Content correctness (button text, link URL, Join AFP text change)
  • CSS styling (background color, flex layout, font hierarchy)
  • Duplicate-init protection (guard against double injection)
  • Sitewide behavior (inner pages served the same header)
  • Responsive breakpoint (button hidden at ≤1199px, visible at ≥1200px)
  • Visual screenshot capture across all 4 desktop browsers

QA Feedback & Issues Found

🐛 BUG — Control Button Text Mismatch (Content)

SeverityHigh — wrong copy ships to visitors
Filelocal_testing/Local2/variation/vB.js — line 31
Figma says (Control)REGISTER FOR AFP 2026
Code has (Control)Register for FP&A Forum ← stale copy from AFP05

The control code was carried forward from the AFP05 winner without updating the button text for AFP 2026. The automated tests passed because TC-04 was written to assert what the code already contained, not what the Figma design specified. The test was matching code against itself, not code against design.

Fix required: Update vB.js line 31 from Register for FP&A Forum → REGISTER FOR AFP 2026, then re-run TC-04.

📋 Process Feedback — Correct QA Order for A/B Tests

The root cause of the missed bug was starting from the code rather than the design. The correct workflow for every A/B test QA is:

  1. Read the Figma design first — extract every piece of expected content (button text, colors, layout, link URLs) directly from the design reference image before looking at any code.
  2. Read the variation/control code — compare what the code does against the Figma spec. Flag any mismatch as a bug before writing a single test.
  3. Verify against the live URL — open the preview/force URL in a browser to confirm the code renders as the Figma intends on the actual site.
  4. Write tests against the Figma spec — test assertions must reflect the design-expected values, not the code's current values. If the code is wrong, the test should fail and surface the bug, not silently pass.
For AFP10 specifically:
Step 1 → Figma (AFP10.png) shows Control button: REGISTER FOR AFP 2026
Step 2 → vB.js line 31 says: Register for FP&A Forum → MISMATCH — bug raised
Step 3 → Live URL: https://www.financialprofessionals.org/ (verify rendered output matches Figma)
Step 4 → TC-04 should assert "REGISTER FOR AFP 2026" (Figma), not "Register for FP&A Forum" (stale code)
